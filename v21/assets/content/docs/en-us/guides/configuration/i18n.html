ngx-gantt supports 6 built-in language packs, and also allows extending custom language packs.

Built-in Language Packs

ngx-gantt includes the following language packs:

Language Identifier Description
Simplified Chinese zh-hans / zh-cn Default language
Traditional Chinese zh-hant / zh-tw Traditional Chinese
English en-us English
Japanese ja-jp Japanese
German de-de Deutsch
Russian ru-ru Русский

Use Built-in Languages

Set the language through global configuration:

import { bootstrapApplication } from '@angular/platform-browser';
import { GANTT_GLOBAL_CONFIG, GanttI18nLocale } from '@worktile/gantt';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent, {
  providers: [
    {
      provide: GANTT_GLOBAL_CONFIG,
      useValue: {
        locale: GanttI18nLocale.enUs // or 'en-us'
      }
    }
  ]
}).catch((err) => console.error(err));

Custom Language Packs

Register a Custom Language Pack

As an example, using French (fr-fr):

Note: The format strings in tickFormats depend entirely on the format function of date-fns. The format rules must follow the date-fns format specification.
import { GanttI18nLocaleConfig, GanttViewType } from '@worktile/gantt';
import { fr } from 'date-fns/locale';

const frenchLocale: GanttI18nLocaleConfig = {
  id: 'fr-fr',
  dateLocale: fr, // date-fns locale
  views: {
    [GanttViewType.hour]: {
      label: 'Vue horaire',
      tickFormats: {
        period: 'd MMM',
        unit: 'HH:mm'
      }
    },
    [GanttViewType.day]: {
      label: 'Vue journalière',
      tickFormats: {
        period: 'MMMM yyyy',
        unit: 'd'
      }
    },
    [GanttViewType.week]: {
      label: 'Vue hebdomadaire',
      tickFormats: {
        period: 'yyyy',
        unit: 'Semaine w'
      }
    },
    [GanttViewType.month]: {
      label: 'Vue mensuelle',
      tickFormats: {
        period: 'yyyy',
        unit: 'MMM'
      }
    },
    [GanttViewType.quarter]: {
      label: 'Vue trimestrielle',
      tickFormats: {
        period: 'yyyy',
        unit: 'T Q'
      }
    },
    [GanttViewType.year]: {
      label: 'Vue annuelle',
      tickFormats: {
        unit: 'yyyy'
      }
    }
  }
};

Register it using GANTT_I18N_LOCALE_TOKEN:

import { bootstrapApplication } from '@angular/platform-browser';
import { GANTT_I18N_LOCALE_TOKEN, GANTT_GLOBAL_CONFIG, GanttI18nLocaleConfig } from '@worktile/gantt';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent, {
  providers: [
    {
      provide: GANTT_I18N_LOCALE_TOKEN,
      useValue: frenchLocale,
      multi: true
    },
    {
      provide: GANTT_GLOBAL_CONFIG,
      useValue: {
        locale: 'fr-fr' // Use the custom language pack ID
      }
    }
  ]
}).catch((err) => console.error(err));

Override Built-in Language Packs

Using the same ID will override a built-in language pack:

import { GanttI18nLocaleConfig, GanttI18nLocale, GanttViewType, zhHansLocale } from '@worktile/gantt';

const customZhHans: GanttI18nLocaleConfig = {
  id: GanttI18nLocale.zhHans, // Use the built-in ID
  views: {
    [GanttViewType.day]: {
      label: 'Day',
      tickFormats: {
        period: 'yyyy-MM',
        unit: 'dd'
      }
    },
    ...
  }
};

import { bootstrapApplication } from '@angular/platform-browser';
import { GANTT_I18N_LOCALE_TOKEN } from '@worktile/gantt';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent, {
  providers: [
    {
      provide: GANTT_I18N_LOCALE_TOKEN,
      useValue: customZhHans,
      multi: true
    }
  ]
}).catch(err => console.error(err));

Common Questions

Q: How do I override text for only specific views?

A: Use the spread operator to expand all existing views first, and then override only the parts you need:
import { GanttI18nLocaleConfig, GanttI18nLocale, GanttViewType, zhHansLocale } from '@worktile/gantt';

const partialLocale: GanttI18nLocaleConfig = {
  id: GanttI18nLocale.zhHans,
  views: {
    ...zhHansLocale.views, // Expand all existing views first
    [GanttViewType.day]: {
      // Then override the parts you need
      label: 'Custom Day View',
      tickFormats: {
        period: 'yyyy-MM',
        unit: 'dd'
      }
    }
    // Other views use default values (kept via spread)
  }
};

Q: Is dateLocale required?

A: It is not required, but it is recommended to provide it to ensure correct date formatting.

Q: Do the format strings not work or show incorrectly?

A: The format strings in tickFormats must follow the date-fns format specification. If the format is incorrect, refer to the date-fns format documentation and verify the format string.
